# Teaching Levels

## Beginner

Start with observable behavior and one concrete example. Define terminology immediately. Explain one function or state change at a time and avoid unexplained shorthand.

## Intermediate

Assume familiarity with variables, functions, classes, and basic control flow. Emphasize module interaction, data ownership, error paths, complexity, and design tradeoffs.

## Advanced

Focus on invariants, contracts, architecture, performance, concurrency, security boundaries, and alternative designs. Skip syntax explanations unless they are language-specific or surprising.

## Adjusting level

Move down a level when the learner cannot predict a small variation. Move up when they accurately trace behavior and explain the relevant concept in their own words. Do not equate vocabulary with understanding.
