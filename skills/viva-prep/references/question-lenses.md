# Question Lenses

Create questions from the real project using these lenses.

## Foundation

- What problem does this project solve?
- Where does execution begin?
- What are the main objects, records, or modules?

## Trace

- Trace one realistic input through the exact functions and branches.
- Which values change, and where are they stored?
- What output appears when a boundary condition is reached?

## Defend

- Why is this data structure, class boundary, or API suitable here?
- What tradeoff did this design make?
- Which rubric requirement is satisfied by this section?

## Troubleshoot

- What happens when data is missing, duplicated, invalid, or unavailable?
- How would you reproduce and isolate a reported failure?
- Which observation would distinguish two possible causes?

## Extend

- What is the smallest safe way to add the requested feature?
- Which tests or modules would be affected?
- What would you redesign if the scale increased substantially?

Prefer questions whose answers can be checked against artifacts. Avoid trivia unrelated to the project or course outcomes.
